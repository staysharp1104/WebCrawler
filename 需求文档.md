# 多平台小说榜单爬取与可视化系统 — 最终完整需求文档

> 版本：v2.0 | 更新：2026-04-15

---

## 1. 项目整体说明

本系统面向**番茄小说**、**飞卢小说**、**七猫小说**、**起点中文网**四大网文平台，实现：

- 新书榜分类 Top20 数据爬取
- 书籍基础信息结构化入库
- 单本书前十章深度爬取、本地文本存储
- 全流程任务管理
- 前端数据可视化
- **每周全自动定时刷新榜单数据**，实现常态化、周期性数据更新

所有业务逻辑严格适配现有五张数据表结构（rank_books、books、chapters、crawl_tasks、chat_history），不新增表、不改字段。

---

## 2. 核心爬取业务需求（基础能力）

### 2.1 爬取范围

| 维度 | 说明 |
|------|------|
| 目标平台 | 番茄（fanqie）、飞卢（feilu）、七猫（qimao）、起点（qidian） |
| 榜单类型 | 各平台官方新书榜 |
| 数据范围 | 新书榜下**所有分类**，每个分类取前 **20 本** |
| 章节深度 | 每本书固定爬取 **第 1–10 章** |

### 2.2 爬取数据字段

完全匹配 `rank_books`、`books` 表字段：

**榜单信息：** 排名、书名、作者、分类标签、阅读人数、书籍状态、封面URL

**书籍主信息：** book_id、标题、作者、简介、分类、字数、总章节数、书籍链接、封面地址

**章节信息：** 前10章章节标题、章节链接、正文内容

### 2.3 章节爬取规则

- 每本上榜书籍固定爬取 **第 1–10 章**
- 章节正文**不存数据库**，保存为本地 `txt` 文件，路径写入 `chapters.content_path`
- 已爬取完成的书籍（`crawl_status=2`）**不再重复爬章节**
- 付费/加密章节自动跳过，记录失败状态，**不阻塞整体任务**

## 3. 每周定时自动刷新需求（新增核心能力）

系统支持**周期性全自动榜单更新**，实现榜单数据常态化迭代，无需人工操作。

### 3.1 定时任务规则

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| 执行周期 | 每周一次 | 自动触发 |
| 默认执行时间 | 每周周日 02:00 | 支持后台自定义周期、时间、开关 |
| 触发方式 | 系统定时触发 + 手动一键刷新 | 双模式 |

### 3.2 每周刷新完整业务流程

```
每周定时任务启动
      │
      ▼
1. 生成批量刷新任务 → crawl_tasks (task_type = rank_weekly_refresh)
      │
      ▼
2. 遍历四大平台所有新书榜分类，重新抓取各分类 Top20
      │
      ▼
3. 新榜单数据增量更新至 rank_books（保留历史快照）
      │
      ▼
4. 榜单书籍数据比对
   ├── 旧书籍 → 更新字数/状态/简介/封面，不重置章节爬取状态
   └── 本周新上榜书籍 → 新增 books，crawl_status=0，生成章节爬取任务
      │
      ▼
5. 自动对「本周新增上榜书籍」执行前十章爬取
      │
      ▼
6. 全部任务状态/重试次数/错误信息/执行结果落库
```

### 3.3 数据更新策略（关键约束）

| 数据表 | 策略 | 说明 |
|--------|------|------|
| **rank_books** | 增量留存 | 每周保留独立榜单快照，不覆盖历史，可追溯每周榜单变化 |
| **books** | 动态更新 | 存量书籍更新信息，保留已爬章节成果 |
| **chapters** | 只增不改 | 仅新书爬取章节，旧书不重复生成文件、不冗余数据 |
| **crawl_tasks** | 全量记录 | 每周刷新日志全量保留，可长期追溯每次刷新结果 |

### 3.4 容错与防风控策略

- 每周刷新自动限流、自动间隔等待，避免高频请求被平台封禁
- **单平台爬取失败不影响其他平台**任务
- 失败任务自动重试 **3 次**，重试间隔 5 分钟
- 所有错误信息精准记录在 `error_msg`，后台可直接查看

## 4. 后台可视化页面完整需求

前端页面全部对接现有五张表，实现数据**可看、可查、可管控、可运维**。

### 4.1 四大核心列表页面

| 页面 | 数据源 | 功能 |
|------|--------|------|
| **榜单数据列表** | rank_books | 展示每周榜单快照、排名、分类、平台、作者、阅读人数，支持按时间/平台筛选 |
| **书籍数据列表** | books | 展示全部书籍、爬取状态标签、字数、章节完成情况、封面预览，支持搜索/筛选/重试 |
| **章节列表** | chapters | 展示章节标题、序号、文件路径、文件大小，支持预览正文、打开文件 |
| **任务监控** | crawl_tasks | 展示所有爬取任务（含每周刷新任务）、失败日志、重试次数、任务状态 |

### 4.2 新增：定时任务配置页面

专门用于管理每周刷新策略：

- **定时任务开关**（开启/关闭每周自动刷新）
- **可配置**：每周执行星期、具体时分
- **一键手动触发**「本周榜单刷新」按钮
- 展示最近 **10 次**每周刷新执行记录、成功/失败统计

### 4.3 数据看板统计

| 指标 | 说明 |
|------|------|
| 总书籍数 | books 表总数 |
| 各平台书籍分布 | 按 source 分组统计 |
| 章节爬取完成率 | crawl_status=2 / 总数 |
| 失败率 | crawl_status=3 / 总数 |
| 本周新增书籍数量 | 本周 created_at 的 books 记录 |
| 本周任务执行成功率 | 本周 crawl_tasks 任务成功/失败比 |

## 5. 数据表字段适配说明（100% 兼容现有表）

本次所有新增「每周刷新」功能**无需新增数据表、无需改字段**，完全复用现有五张表：

| 数据表 | 兼容策略 |
|--------|---------|
| **crawl_tasks** | 通过 `task_type = rank_weekly_refresh` 区分每周刷新任务 |
| **rank_books** | 自动积累每周榜单快照（`created_at` 时间戳天然区分周次） |
| **books** | 动态更新书籍信息，保留章节爬取进度 |
| **chapters** | 仅增量新增新书章节 |
| **chat_history** | 保留扩展能力，不影响主业务 |

## 附录 B：每周刷新与普通爬取对比

| 维度 | 普通爬取（--rank-only） | 每周刷新（定时任务） |
|------|------------------------|---------------------|
| 触发方式 | 手动执行 | 定时自动 + 手动一键 |
| 榜单数据 | 全量重取 | 增量留存，保留历史快照 |
| 已爬书籍 | 重置状态可能重复爬 | 保留章节成果，不重复爬 |
| 新书上榜 | 不区分新旧 | 自动识别，自动爬章节 |
| 任务类型 | crawl_rank | rank_weekly_refresh |
| 容错 | 单次失败即退出 | 逐平台独立，3次重试 |
